使い方ガイド

5 ステップで投稿が完成します

ステップ

1
GitHub でログイン

code-tweet.vercel.app を開き、「GitHub でログイン」を押します。

ログインなしでも 3 回までは試せます（README 上限 1,000 文字）。

2
リポジトリを選ぶ

連携した GitHub アカウントから、宣伝したいリポジトリを選択します。

プライベートリポジトリを対象にする場合は Pro プランの契約と Personal Access Token の登録が必要です。

3
投稿を生成

必要に応じてプロジェクト説明・トーン・ハッシュタグを補足し、「投稿を生成」ボタンを押します。AI が 5 件の投稿案を返します。

4
気に入った案を選ぶ

5 件から好みの 1 つを選んでコピー。各案には推奨投稿時刻と「なぜその時刻か」の理由も添えられます。

5
X に投稿

「X で投稿」ボタンから X の投稿画面に遷移し、内容を確認して投稿します。

よくある質問

無料と Pro の違いは？

無料は月 3 回・README 1,000 文字まで・公開リポジトリのみ。Pro は月 30 回・README 10,000 文字まで・プライベートリポジトリも対象です。価格は ¥980 / 月です。

プライベートリポジトリを読ませるには？

Pro プランに加入のうえ、設定ページから GitHub の Fine-grained Personal Access Token（Contents: Read-only 権限）を登録してください。詳しい手順は GitHub の README に記載しています。

生成回数のリセットはいつ？

無料・Pro どちらも月単位でリセットされます。詳細はアプリ内の利用状況表示で確認できます。

生成された投稿は誰かに見られますか？

生成内容はあなたのアカウントに紐づいて保存され、他のユーザーには公開されません。詳しくはプライバシーポリシーをご覧ください。

エラーが出て修正できません

入力中の内容を保持したまま画面下部にエラーが表示されます。文字数オーバーなど条件を満たしていない場合は、メッセージに従って修正してください。それでも解決しない場合はお問い合わせまたはGitHub Issuesからご連絡ください。
