使い方はたった 3 ステップ

面倒な設定は一切なし。リポジトリ URL を貼るだけ。

1

URL を貼る

紹介したい GitHub リポジトリの URL を入力欄に貼り付けます。

2

条件を選ぶ

トーン（真面目／カジュアル）やターゲット読者を指定。空欄でも OK。

3

5 本まとめて生成

紹介・使い方・ユースケース・共感・CTA の 5 本が一度に出来上がり。

継続発信に効くこだわり

📖 README を自動取得

URL を貼るだけでリポジトリの README を自動で読み込み、AI が要点を抽出します。

🎯 5 段ストーリーアーク

紹介 → 使い方 → ユースケース → 共感 → CTA の流れで 5 本生成。1 週間分の投稿計画がそのまま完成。

🕒 推奨投稿日時つき

各ポストに「いつ投稿すると良いか」の目安が付くので、投稿予約にそのまま使えます。

✂️ 280 字自動チェック

X の文字数制限に収まるかリアルタイムで確認。コピペしてすぐ投稿できます。

🚀 ワンクリックで X へ

生成したポストは「X で投稿」ボタンから、本文入りの投稿画面をそのまま開けます。コピー&貼り付け不要。

New

App SS — App Store のスクショも、同じ README から

Code Tweet と同じ README を使って、App Store 提出用のスクリーンショット（＝ダウンロードを左右する“広告”）をAIと一緒に作成できるツールです。README を貼るだけでアプリ名・説明・機能をAIが自動で読み取り、見出しコピー・このアプリ専用の配色パレット・レイアウトまで一気に生成。あとは自分のアプリ画面を入れて書き出すだけ、すべてブラウザだけで完結します。

📖 README からアプリ情報を自動入力

README を貼り付けるだけで、AIがアプリ名・説明・主な機能を読み取って入力欄に反映。仮入力として使い、そこから自由に編集できます。

🎨 専用配色パレットをAIが都度生成

既製テーマから選ぶだけでなく、アプリの雰囲気に合わせた配色をAIがそのつど新しく生成。テーマ 11 種・日本語フォント 9 種から手動でも選べます。

✂️ 強調ワードを自動ハイライト

各見出しで一番刺さる語をAIが選び、アクセント色で自動強調。プロのスクショ広告に欠かせない“一語の強調”をそのまま再現します。

🔒 画像は端末内だけで処理

取り込んだスクリーンショットはブラウザ内にのみ保存され、サーバーには送信されません。未公開アプリのスクショも安心して扱えます。

🚀 App Store 全サイズを一括書き出し

6.9 / 6.5 / 6.3 / 6.1 インチを一度に PNG で書き出し、そのまま App Store Connect にアップロードできます。
