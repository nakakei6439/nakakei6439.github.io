😩

毎日ネタを考えるのが大変

「同じ告知ばかり」「もう何を書けばいいか…」となって発信が止まる。

📝

文章を考える時間がない

開発に集中したい。でも宣伝もしないと使ってもらえない。

📉

毎回トーンがバラバラ

真面目だったりカジュアルだったり、ブランドが定まらない。

使い方はたった 3 ステップ

面倒な設定は一切なし。リポジトリ URL を貼るだけ。

1

URL を貼る

紹介したい GitHub リポジトリの URL を入力欄に貼り付けます。

2

条件を選ぶ

トーン（真面目／カジュアル）やターゲット読者を指定。空欄でも OK。

3

5 本まとめて生成

紹介・使い方・ユースケース・共感・CTA の 5 本が一度に出来上がり。

継続発信に効くこだわり

📖 README を自動取得

URL を貼るだけでリポジトリの README を自動で読み込み、AI が要点を抽出します。

🎯 5 段ストーリーアーク

紹介 → 使い方 → ユースケース → 共感 → CTA の流れで 5 本生成。1 週間分の投稿計画がそのまま完成。

🕒 推奨投稿日時つき

各ポストに「いつ投稿すると良いか」の目安が付くので、投稿予約にそのまま使えます。

✂️ 280 字自動チェック

X の文字数制限に収まるかリアルタイムで確認。コピペしてすぐ投稿できます。
